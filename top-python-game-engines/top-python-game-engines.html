Обзор игровых движков на Python

Игровые движки для Python чаще всего принимают форму библиотек Python, которые могут быть установлены различными способами. Большинство из них доступны на PyPI и могут быть установлены с помощью pip. Однако некоторые из них доступны только на GitHub, GitLab или в других местах общего доступа к коду, и для их установки могут потребоваться другие шаги. В этой статье будут рассмотрены методы установки для всех обсуждаемых движков.

Python - это язык программирования общего назначения, и он используется для различных задач, помимо написания компьютерных игр. Напротив, существует множество различных автономных игровых движков, специально предназначенных для написания игр. Некоторые из них включают:

  • The Unreal Engine
  • Unity
  • Godot

Эти специально разработанные игровые движки отличаются от Python-игровых движков следующими клюбевыми аспектами:

  • Поддержка языка. Такие языки, как C++, C# и JavaScript, популярны для игр, написанных на автономных игровых движках, поскольку сами движки часто написаны на этих языках. Очень немногие автономные движки поддерживают Python.
  • Проприетарная поддержка сценариев: К тому же, многие автономные игровые движки поддерживают и поддерживают свои собственные языки сценариев, которые могут не походить на Python. Например, Unity изначально использует C#, в то время как Unreal лучше всего работает с C++.
  • Поддержка платформы: Многие современные автономные игровые движки могут создавать игры для различных платформ, включая мобильные и специализированные игровые системы, с минимальными усилиями. Напротив, перенос игры на Python на различные платформы, особенно на мобильные платформы, может оказаться серьезной задачей.
  • Варианты лицензирования: Игры, написанные с использованием автономного игрового движка, могут иметь различные варианты лицензирования и ограничения в зависимости от используемого движка.

Итак - зачем вообще использовать Python для написания игр? Одним словом - это Питон! Использование автономного игрового движка часто требует от вас изучения нового языка программирования или написания сценариев. Игровые движки Python используют ваши существующие знания Python, сокращая время обучения и помогая вам быстро продвигаться вперед.

Существует множество игровых движков, доступных для среды разработки Python. Все движки, о которых вы узнаете здесь, соответствуют следующим критериям:

  • Они являются относительно популярными движками или охватывают аспекты игр, которые обычно не освещаются.
  • В настоящее время они поддерживаются в рабочем состоянии.
  • У них есть хорошая документация.

Для каждого движка вы изучите:

  • Методы инсталляции
  • Основные понятия, а также допущения, котрые делает движок
  • Основные особенности и возможности
  • Две игровые реализации, позволяюшие их сравнивать

Pygame::before

Когда люди думают об игровых движках Python, первая мысль, которая приходит многим в голову, - это Pygame. На самом деле, на сайте Real Python уже есть отличный учебник по Pygame.

Написанный как замена остановленной библиотеки PySDL, Pygame обертывает и расширяет библиотеку SDL, которая расшифровывается как Simple DirectMedia Layer. SDL обеспечивает кроссплатформенный доступ к базовым компонентам мультимедийного оборудования вашей системы, таким как звук, видео, мышь, клавиатура и джойстик. Кроссплатформенность SDL и Pygame означает, что вы можете писать игры и мультимедийные программы на Python для любой платформы, которая их поддерживает!

Установка Pygame::before

Pygame доступна на PyPI, поэтому после создания и активации виртуальной среды вы можете установить ее с помощью соответствующей команды pip: 

            
            
                (venv)$ python -m pip install pygame

Как только это будет сделано, вы можете проверить установку, запустив пример, прилагаемый к библиотеке: 

            
            
                (venv) $ python -m pygame.examples.aliens

Теперь, когда вы установили Pygame, вы можете начать использовать его прямо сейчас. Если вы столкнетесь с проблемами во время установки, то в руководстве по началу работы описаны некоторые известные проблемы и возможные решения для всех платформ.
